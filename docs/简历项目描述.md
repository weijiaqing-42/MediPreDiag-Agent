**MediPreDiag-Agent — 基于多智能体协作的医疗预诊系统** | 个人项目 2026.04-2026.05

**项目背景**
通用大模型在医疗场景中存在幻觉率高、缺乏专业知识溯源、无法区分症状严重程度等问题，且单一 Agent 难以覆盖症状分析、药品咨询、医院推荐等多样化预诊需求，缺乏基于权威数据集的量化评测手段。

**主要工作**
- **多智能体编排与评测**：基于 LangGraph 有向图实现 6 个专职 Agent（意图分类、症状提取、严重程度评估、医疗建议生成、药品问答、地点推荐），通过 Supervisor 统一调度与条件边动态路由，设计业务中断/回滚/超时降级机制；在 MedQA USMLE Step 2 CK 数据集上临床诊断准确率达到 74%（50 题英文临床选择题），在 MedAgentBench v2 上 Task Success Rate 达到 83.33%（30 项 EHR 模拟任务）；
- **RAG 知识库与混合检索**：构建 Milvus 向量检索 + BM25 (jieba) 稀疏检索双路并行 → RRF 融合 → DashScope qwen3-rerank 重排序的四级检索管线，引入 Content MD5 去重解决 RRF 融合后文档重复问题；覆盖 125 篇医学文献（症状知识、药品信息、疾病科普、临床指南、检查解读 5 类 × 25 篇）；
- **意图识别容错机制**：基于 qwen-turbo 的 Few-shot 意图分类器，支持 6 类意图（症状/药品/地点/紧急/闲聊/未知），实现三层容错 JSON 提取（代码块提取 → 整段解析 → 正则匹配）与字段校验，输出格式合规率 100%；、、
- **两层记忆系统**：短期记忆基于 Redis (TTL=30min) 存储会话上下文与中断状态；长期记忆基于 Milvus 向量化存储用户历史诊断摘要，异步 LLM 总结压缩降低存储开销；MySQL 持久化用户档案与诊断记录；
- **G-Eval 质量评估**：基于 MCP 协议封装高德地图 POI API，根据严重程度自动推荐附近药店/门诊/急诊；基于 G-Eval 框架从安全性（4.60/5）、专业性（3.80/5）、有用性（2.73/5）、流畅度（4.87/5）四个维度对 Agent 对话进行 LLM Judge 评分，综合均分达到 4.00/5。
