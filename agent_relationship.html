🧩 Agent 节点详情

🧠
memory_loader
Redis + Milvus
系统入口节点。加载用户短期记忆（Redis，TTL=30min）和长期健康摘要（Milvus 向量检索），为后续流程提供个性化上下文。
🎯
intent_classifier
qwen-turbo
快速意图分类节点。将用户输入分为：symptom_query / drug_query / location_search / emergency / chitchat / unknown，决定后续路由分支。
🔬
symptom_analysis
qwen-vl-max + qwen3-max
症状分析复合节点。先调用 image_analyzer (qwen-vl-max) 做多模态图片分析，再调用 symptom_extractor (qwen3-max) 结构化提取症状和可能的疾病。
📚
rag_retriever
Milvus + BM25 + reranker
RAG混合检索节点。Milvus向量检索（HNSW）+ BM25稀疏检索，经RRF融合后由 bge-reranker-v2-m3 重排序，Top-5 结果输入后续LLM上下文。
⚡
severity_evaluator
规则引擎 + qwen-turbo
严重程度评估节点。规则优先匹配危险关键词（胸痛/呼吸困难→severe），LLM兜底评估复杂症状。输出 mild / moderate / severe / unknown 驱动后续路由。
🏥
medical_advisor
qwen3-max
医疗建议生成节点。结合 RAG检索知识、长期记忆摘要、短期对话历史，用 qwen3-max 生成个性化医疗建议（症状分析→措施建议→危险信号→免责声明）。
💊
drug_qa_agent
qwen3-max + RAG
药品问答Agent。提取药品名称和问题类型（用法/禁忌/副作用/剂量），检索药品说明书 + RAG知识库，由 qwen3-max 生成专业回答。
📍
location_recommender
高德地图 API
地点推荐Agent。根据严重程度自动选择搜索策略：mild→药店(1km) / moderate→综合医院(3km) / severe→急救中心(5km)。调用高德POI周边搜索MCP Tool。
📝
response_synthesizer
qwen-turbo
最终响应合成器。根据意图类型整合各Agent结果：紧急→急救提醒 / 症状→医疗建议+地点 / 药品→专业回答 / 地点→POI列表 / 闲聊→友好引导。
💾
memory_updater
Redis + Milvus + MySQL
记忆更新节点。保存对话到Redis短期记忆，累计6轮后生成摘要向量化存入Milvus长期记忆，同时写入MySQL诊断记录。流程终点。
⚠️
interrupt_handler
中断/回滚/超时处理
异常处理节点。处理用户主动取消、API超时(15s)、工具调用异常、置信度过低等场景。支持状态快照回滚、降级回复、追问生成。
